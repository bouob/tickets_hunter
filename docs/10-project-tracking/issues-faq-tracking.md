## 按平台分類

### TixCraft 拓元

#### Issue #173: 瀏覽器異常頁面
**問題**: 出現「瀏覽器異常」頁面，程式停擺
**原因**: 刷新頻率過快，觸發 TixCraft 反爬蟲機制
**解決方案**:
1. 調高「自動刷新間隔秒數」至 3-5 秒
2. 等待 1-2 分鐘後重試
3. 避免同時開啟多個搶票視窗

**設定位置**: 進階設定 → 自動刷新間隔秒數 (auto_reload_page_interval)

---

#### Issue #158, #144, #137, #63, #32: Cookie SID 登入失敗
**問題**: 設定 TIXUISID 後無法自動登入
**原因**:
- Cookie 設定後需要重啟程式才會生效
- TixCraft 在 2024/11/28 變更 Cookie 機制

**解決方案**:
1. 填入 TIXUISID
2. 點擊「儲存」
3. **點擊「結束」關閉程式** (關鍵步驟)
4. 重新開啟程式
5. 點擊「搶票」

**社群貢獻** (@foxliu0328):
> "填入 TIXUISID 後，點存檔 → 點結束 → 再重新開啟設定檔 → 再點搶票，就可以自動登錄了"

**注意**: 使用 v2025.11.30 以上版本

---

#### Issue #166: 門票數量/驗證碼未自動填寫
**問題**: 進入訂票頁面後，票數和驗證碼沒有自動填寫
**原因**: NoDriver 引擎不穩定或頁面載入較慢
**解決方案**: 啟用「區域自動遞補」功能

**社群貢獻** (@a00user00name):
> "我用 nodriver 也會這樣，不過打開**區域自動遞補**就會正常了"

### UDN 聯合報售票網

#### FAQ: UDN 登入需要手動操作
**問題**: UDN 登入時程式停在登入頁面，需要手動操作

**原因**: UDN 使用 Google reCAPTCHA 驗證，程式無法自動完成

**解決方案**: ⚠️ **UDN 登入是半自動的**
1. 程式會自動填寫帳號密碼
2. **使用者需手動完成 reCAPTCHA 圖片驗證**（選擇公車、腳踏車、紅綠燈等圖片）
3. **使用者需手動點擊「登入」按鈕**
4. 程式會自動偵測登入成功並繼續

**設定位置**: 進階設定 → UDN 帳號/密碼

**技術說明**:
- reCAPTCHA 是 Google 的人機驗證服務
- 設計目的就是防止自動化程式
- 這是平台安全措施，非程式 Bug

**日誌訊息**: 當看到以下訊息時，請手動完成驗證：
```
[UDN LOGIN] Credentials filled, waiting for user to complete reCAPTCHA and login...
```

## 按問題類型分類

### 環境與安裝問題

#### Issue #7: ARM 環境 / ddddocr 無法使用
**問題**: 出現「ddddocr 組件無法使用，您可能在 ARM 環境下運行」
**原因**:
- Python 版本過新（3.13+）
- ARM 環境（Apple Silicon Mac）
- onnxruntime 安裝問題

**解決方案**:
1. **降級 Python 版本**（推薦）: 使用 Python 3.10
2. **Apple Silicon Mac**: 使用 Rosetta 執行
   ```bash
   arch -x86_64 python3 nodriver_tixcraft.py --input settings.json
   ```
3. **重新安裝 OCR 套件**:
   ```bash
   pip uninstall ddddocr onnxruntime -y
   pip install ddddocr
   ```

---

#### Issue #60, #59, #55: 程式無法啟動 / 不是內部或外部命令
**問題**: 執行時出現「不是內部或外部命令」錯誤
**原因**: 環境變數或路徑問題
**解決方案**:
1. 使用 exe 執行檔（推薦）
2. 確認 Python 已加入 PATH
3. 從正確目錄執行
